# Stored Procedures and Functions

## Functions vs Procedures

| | Function | Procedure |
|---|---|---|
| Returns value | Yes (mandatory) | Optional (via OUT params) |
| Used in SELECT | Yes | No |
| Transaction control | No (cannot COMMIT/ROLLBACK) | Yes |
| SQL standard | Yes | Yes (SQL:1999+) |

## Creating a function (PostgreSQL)

```sql
CREATE OR REPLACE FUNCTION get_customer_total(p_customer_id INT)
RETURNS NUMERIC AS $$
    SELECT COALESCE(SUM(oi.quantity * oi.unit_price), 0)
    FROM orders AS o
    JOIN order_items AS oi ON o.id = oi.order_id
    WHERE o.customer_id = p_customer_id;
$$ LANGUAGE sql STABLE;
```

Usage:
```sql
SELECT first_name, get_customer_total(id) AS total_spent
FROM customers;
```

## PL/pgSQL functions

For procedural logic (IF, LOOP, variables):

```sql
CREATE OR REPLACE FUNCTION classify_customer(p_id INT)
RETURNS TEXT AS $$
DECLARE
    total NUMERIC;
BEGIN
    SELECT get_customer_total(p_id) INTO total;

    IF total > 500 THEN
        RETURN 'VIP';
    ELSIF total > 100 THEN
        RETURN 'Regular';
    ELSE
        RETURN 'New';
    END IF;
END;
$$ LANGUAGE plpgsql STABLE;
```

## Stored procedures (PostgreSQL 11+)

```sql
CREATE OR REPLACE PROCEDURE restock_product(p_id INT, p_quantity INT)
LANGUAGE plpgsql AS $$
BEGIN
    UPDATE products
    SET stock = stock + p_quantity
    WHERE id = p_id;

    INSERT INTO audit_log (action)
    VALUES ('Restocked product ' || p_id || ' by ' || p_quantity);

    COMMIT;
END;
$$;

CALL restock_product(1, 100);
```

## Dropping

```sql
DROP FUNCTION get_customer_total(INT);
DROP PROCEDURE restock_product(INT, INT);
```
